# Vibe Coding Workflow Skill

이 스킬은 코드를 변경하고 제안할 때 **모든 AI 어시스턴트**가 반드시 준수해야 하는 공통 워크플로우를 담고 있습니다. 무분별한 코드 생성으로 인한 런타임 오류나 시스템 파편화를 막기 위한 안전장치입니다.

## 🛑 Phase 1: 아키텍처 파악 (Pre-Check)
1. **요구사항 수신**: 사용자가 특정 파일이나 기능의 수정을 요청합니다.
2. **의존성 확인**: 바로 코드를 수정하지 말고, 대상 파일이 어디서 호출되고 있는지 확인합니다. 프로젝트에 제공된 `impact_map` 스킬의 명령어(`architecture_mapper.py` 등)를 활용하십시오.

## ✍️ Phase 2: 코드 작성 및 수정 (Execution)
1. 사전 파악된 제한된 범위와 의존성을 고려하여 코드를 수정합니다.
2. 만약 함수 시그니처나 인터페이스, DB 스키마가 변경되었다면, 그것을 참조하는 다른 의존성 모듈들 또한 함께 수정해야 함을 잊지 마십시오.
3. 변경 확정 및 로컬 저장(Commit)을 수행합니다.

## 🔍 Phase 3: 변경 영향도 자가 검증 (Validation)
1. **Impact Analyzer 실행**: `impact_map` 스킬을 활용하여 `impact_analyzer.py`를 실행하세요.
2. **리포트 제출**: 스크립트가 생성한 Report 목록을 읽고, 위험 수준이 `High`(고위험)로 나오면 사용자에게 중대한 결함이나 부작용이 발생할 수 있음을 경고한 후 다음 단계로 넘어가세요.

## 📌 명심할 점 (System Directive)
당신은 코드를 작성/수정하기 전에 사용자에게 "제가 코드를 짜기 전에 먼저 아키텍처를 스캔하겠습니다"라고 선임 엔지니어처럼 명확하게 응답하여 신뢰감을 주어야 합니다. 변경된 파일 목록과 예상 위험도를 항상 요약 리포트로 제출하세요.
